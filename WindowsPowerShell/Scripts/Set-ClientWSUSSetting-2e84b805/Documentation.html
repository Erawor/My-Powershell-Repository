This function allows you to configure one or more system's automatic updates client by manipulating the registry settings assoicated with each setting. It is recommended that you run this with an account that has Administrative rights to each client to avoid any potential errors.

Updated 28 Mar 2014

  • Fixed bug in ScheduleInstallDay parameter
  • Fixed bug in an If statement

 

This script will be used in the January 16th Hey, Scripting Guy blog.

 

Examples:

Set-ClientWSUSSetting -UpdateServer "http://testwsus.com" -UseWSUSServer Enable -AllowAutomaticUpdates Enable -DetectionFrequency 4 -Options DownloadOnly

Configures the local computer to enable automatic updates and use testwsus.com as the update server. Also sets the update detection frequency to occur every 4 hours and only downloads the updates.

Set-ClientWSUSSetting -UpdateServer "http://testwsus.com" -UseWSUSServer Enable -AllowAutomaticUpdates Enable -DetectionFrequency 4 -Options DownloadAndInstall -RebootWarningTimeout 15  -ScheduledInstallDay Monday -ScheduledInstallTime 20

Configures the local computer to enable automatic updates and use testwsus.com as the update server. Also sets the update detection frequency to occur every 4 hours and performs the installation automatically every Monday at 8pm and configured to reboot 15 minutes (with a timer for logged on users) after updates have been installed.
