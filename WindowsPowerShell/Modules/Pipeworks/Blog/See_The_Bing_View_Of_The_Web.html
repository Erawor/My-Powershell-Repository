PowerShell  Pipeworks has a very handy command that you can use to see the web through the eyes of a search engine: Get-Web.

Get-Web helps you consume web pages and build sites that rely on aggregated web data, like small search engines and content aggregation sites.

One handy parameter, -AsMicrodata, helps you pick out Microdata from sites. In case you don't know about it, Microdata is a way that you can embed structred information inside of a web page. Microsoft, Google, and Yahoo have combined forces to help determine a set of common structures for web sites to use, and have posted them on Schema.org. Get-Web -AsMicrodata will parse out that Microdata, which allows you to see the world like a search engine does.

For instance, this pulls out information about a movie on  IMDB

            
Get-Web -Url "http://www.imdb.com/title/tt01/" -AsMicrodata -ItemType http://schema.org/Movie

This one liner can help make me dinner:

Get-Web -Url "http://www.myrecipes.com/recipe/veal-and-artichoke-stew-with-avgolemono-10000000226585/" -AsMicrodata -ItemType http://data-vocabulary.org/Recipe

Having something like Get-Web in your toolkit can help you make mincemeat of content aggregation problems.  Many more sites are adopting Microdata by the day, and you can easily consume, search, and aggregate that content with PowerShell Pipeworks.
