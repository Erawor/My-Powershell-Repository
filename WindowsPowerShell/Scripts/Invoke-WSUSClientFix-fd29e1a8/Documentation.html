This function will allow you to perform a WSUS Client ID fix on both local and remote systems. Sometimes a client will not communicate properly with the WSUS server to due an invalid clientID caused by a system that has been cloned or an imaged created from it without being sysprepped.

This function will perform a registry cleanup as well as removing some files and folders and then forces a check-in against the WSUS server.

Updated 28Mar2014

  • Bug fix in removing files and folders

 

Related blog post: http://learn-powershell.net/2013/10/16/fix-wsus-client-id-using-powershell/

 

Remember to dot source the script before using function!

 

PowerShell
Edit|Remove
powershell 
. .\Invoke-WSUSClientIDFix.ps1
. .\Invoke-WSUSClientIDFix.ps1
 

 

 

PowerShell
Edit|Remove
powershell 
Invoke-WSUSClientIDFix -Computername 'Server' -Verbose
Invoke-WSUSClientIDFix -Computername 'Server' -Verbose
